# CLAUDE.md

이 파일은 Claude Code(claude.ai/code)가 이 저장소에서 작업할 때 필요한 가이드라인을 제공합니다.

## 프로젝트 맥락
이 작업공간은 PM 및 기획 업무를 위해 설계되었으며, IT와 Web3 프로덕트 매니지먼트, UX/UI 전략, 기술 명세서 개발에 중점을 둡니다.

## 핵심 기획 방법론

### 요구사항 분석 프레임워크
- **Goal-Persona-Features (GPF) 분석**: 세부 기획 전에 항상 명확한 프로젝트 목표, 타겟 페르소나, 핵심 기능을 설정
- **MoSCoW 우선순위 설정**: Must-have(필수), Should-have(중요), Could-have(선택), Won't-have(제외) 프레임워크 활용
- **MVP 우선 접근법**: 기능 확장 전 최소 기능 제품(MVP) 범위 식별

### 문서 구조 표준
- **서비스 정의서(SDD)**: 요약, 비즈니스 목표, 성공 지표
- **상세 기능 명세서(PRD)**: 상세 기능 사양, 사용자 스토리, 승인 기준
- **사용자 플로우 문서**: 의사결정 지점과 오류 케이스를 포함한 단계별 사용자 여정 매핑
- **기술 명세서**: 백엔드 데이터 엔티티, API 요구사항, 시스템 아키텍처 고려사항

### 리스크 평가 프로토콜
- **기술적 부채 분석**: 잠재적 기술 과제 및 완화 전략 식별
- **비즈니스 리스크 평가**: 시장 경쟁, 사용자 채택 장벽, 확장성 우려사항
- **구현 가능성**: 자원 요구사항, 일정 제약, 의존성 매핑

## 기획 산출물 형식

### 사용자 스토리 템플릿
```
[페르소나]로서 [목표/혜택]을 위해 [액션]을 원한다.
승인 기준:
- [상황]에서 [액션]을 할 때, [예상 결과]가 나타난다
```

### 마일스톤 기획 구조
- **단계 정의**: 명확한 범위와 목표
- **산출물 명세**: 구체적인 결과물과 성공 기준
- **의존성 매핑**: 전제조건 및 차단 요소
- **리스크 완화**: 비상 계획 및 대안 접근법

## 협업 가이드라인

### 이해관계자 커뮤니케이션
- **경영진 요약**: 비즈니스 임팩트 우선, 기술적 세부사항 후속
- **개발자 인수인계**: 엣지 케이스를 포함한 구현 준비 완료 명세서
- **디자인 협업**: 와이어프레임 요구사항과 함께 사용자 여정 맥락 제공

### 문서화 표준
- **버전 관리**: 주요 명세서 업데이트에 대한 근거와 함께 변경 로그 유지
- **추적성**: 요구사항을 비즈니스 목표 및 사용자 니즈에 연결
- **검토 프로세스**: 최종화 전 기술적 실현 가능성 검증 포함

## Web3 및 블록체인 고려사항
- **스마트 컨트랙트 상호작용**: 온체인 vs 오프체인 데이터 및 로직 분리 정의
- **가스 최적화**: UX 디자인 결정에서 트랜잭션 비용 고려
- **탈중앙화 트레이드오프**: 사용자 경험과 탈중앙화 원칙 간 균형
- **보안 프레임워크**: 지갑 통합, 개인키 관리, 트랜잭션 서명 플로우 처리
