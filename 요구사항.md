# 미션앤리워드 플랫폼 - 기능 요구사항 명세서

## 프로젝트 개요

### 서비스 정의
USDT 리워드 기반 게임 미션 플랫폼으로, 사용자가 출석체크와 게임 미션 완료를 통해 암호화폐 리워드를 획득할 수 있는 웹 애플리케이션

### 비즈니스 목표
- 사용자 일일 활동 유도를 통한 플랫폼 참여도 증대
- 게임화 요소를 통한 사용자 리텐션 향상
- USDT 리워드 시스템을 통한 실질적 가치 제공

### 타겟 페르소나
- **주요 사용자**: 모바일 게임을 즐기며 암호화폐에 관심 있는 20-40대
- **사용 맥락**: 일상 중 짧은 시간 투자로 리워드 획득을 원하는 사용자
- **기술 수준**: 기본적인 모바일 앱 사용 가능, 암호화폐 기초 지식 보유

## 페이지별 기능 요구사항

### 1. 홈 대시보드 (index.html)

#### 1.1 USDT 잔고 표시
**사용자 스토리**: 플랫폼 사용자로서 현재 보유한 USDT 잔고를 확인하여 리워드 누적 현황을 파악하고 싶다.

**승인 기준**:
- 홈 진입 시 실시간 USDT 잔고가 표시된다
- 기본 잔고(2,353.55) + 획득 USDT 합계로 계산된다
- 천단위 콤마와 소수점 2자리 형식으로 표시된다
- 원화 환산액이 함께 표시된다 (1 USDT = 1,350원)

#### 1.2 게임 캐러셀
**사용자 스토리**: 플랫폼 사용자로서 참여 가능한 게임 목록과 각 게임의 미션 진행 상태를 한눈에 파악하고 싶다.

**승인 기준**:
- 10개 게임이 좌우 스와이프 가능한 캐러셀로 표시된다
- 지원 게임 목록: Squishy Cat, Tab Tab Jello, Merge Cat, Hook & Gold, Rich Match, Soda King, Cash Pang, Fate War, Magic Squad, Battle Ship
- 각 게임별 진행 상태가 텍스트로 표시된다:
  - "미션 진행전" (회색)
  - "X/3 미션 진행 중" (회색)  
  - "미션 완료" (녹색)
  - "리워드 받음" (녹색)
- 게임 카드 클릭 시 해당 게임 상세 페이지로 이동한다
- Hook & Gold는 기본 완료 상태로 고정 처리된다

#### 1.3 미션 완료 보상 배너
**사용자 스토리**: 플랫폼 사용자로서 완료된 게임 미션이 있을 때 보상을 받을 수 있다는 알림을 받고 즉시 보상 페이지로 이동하고 싶다.

**승인 기준**:
- 상태 3(미션 완료)인 게임이 1개 이상일 때 하단에 플로팅 배너가 나타난다
- 배너 위치는 화면 하단 90px에 고정된다
- 완료된 게임 수 × 2 USDT 보상 금액이 표시된다
- "받기" 버튼 클릭 시 daily_mission.html로 이동한다
- 슬라이드업/다운 + 페이드 애니메이션이 적용된다 (0.3초)

### 2. 일일 미션 화면 (daily_mission.html)

#### 2.1 출석체크 시스템
**사용자 스토리**: 플랫폼 사용자로서 매일 출석체크를 통해 연속 출석 보상을 받아 지속적인 참여 동기를 얻고 싶다.

**승인 기준**:
- 5일 연속 출석체크 주기로 운영된다
- 일별 보상 구조:
  - 1일: 1 USDT
  - 2일: 1 USDT  
  - 3일: 럭키볼 1개
  - 4일: 1 USDT
  - 5일: 럭키볼 3개
- 출석체크 완료 시 체크마크 또는 럭키볼 이미지로 표시된다
- "[테스트] 내일로" 버튼으로 다음날 시뮬레이션이 가능하다

#### 2.2 게임 미션 프로그레스 바
**사용자 스토리**: 플랫폼 사용자로서 게임 3개 완료 미션의 진행 상황을 시각적으로 확인하고 완료한 게임 순서를 파악하고 싶다.

**승인 기준**:
- 게임 3개 미션 완료가 목표로 설정된다
- 완료 시각 기준 오름차순으로 게임 순서가 표시된다
- Hook & Gold는 항상 첫 번째 위치에 고정된다
- 프로그레스 단계:
  - 0개: 0%, 플레이스홀더 "1", "2" 표시
  - 1개: 33%, 첫 번째 게임 로고 표시
  - 2개: 66%, 두 번째 게임 로고 표시
  - 3개: 100%, 럭키볼 3개 보상 자동 지급

#### 2.3 럭키볼 뽑기 시스템  
**사용자 스토리**: 플랫폼 사용자로서 보유한 럭키볼을 사용하여 랜덤 USDT 리워드를 받고 연속으로 뽑기를 진행하고 싶다.

**승인 기준**:
- 1-3 USDT 범위의 랜덤 보상이 지급된다
- 뽑기 실행 시 보상 금액과 남은 럭키볼 개수가 오버레이로 표시된다
- 럭키볼이 남아있으면 "더 뽑기" 옵션이 제공된다
- UI에서 개수는 검정색, "뽑기"는 녹색으로 표시된다
- USDT 아이콘 + 보상 금액 + 남은 개수가 오버레이에 표시된다

#### 2.4 타이머 시스템
**사용자 스토리**: 플랫폼 사용자로서 일일 미션 리셋까지 남은 시간을 실시간으로 확인하여 미션 완료 시점을 계획하고 싶다.

**승인 기준**:
- 매일 자정(00:00:00)까지 남은 시간을 표시한다
- HH:MM:SS 남음 형식으로 표시된다
- 1초 간격으로 실시간 업데이트된다
- 자정이 지나면 새로운 24시간 카운트다운이 시작된다

### 3. 리워드 내역 화면 (reward_history.html)

#### 3.1 리워드 잔고 표시
**사용자 스토리**: 플랫폼 사용자로서 현재 보유한 USDT와 럭키볼 개수를 확인하여 자산 현황을 파악하고 싶다.

**승인 기준**:
- USDT 잔고가 소수점 2자리까지 표시된다
- 원화 환산액이 함께 표시된다 (1 USDT = 1,350원)
- 럭키볼 개수가 정수로 표시된다 (뽑기 기능은 daily_mission에서만 제공)

#### 3.2 필터 시스템
**사용자 스토리**: 플랫폼 사용자로서 리워드 내역을 원하는 기간과 정렬 순서로 조회하여 거래 패턴을 분석하고 싶다.

**승인 기준**:
- 기본값: 1개월, 최신순으로 설정된다
- 조회 기간 옵션: 1개월, 3개월, 6개월, 직접입력
- 정렬 옵션: 최신순, 과거순
- 바텀시트 형태로 화면 하단에서 슬라이드업 된다
- 최대 폭 480px, 화면 중앙 배치로 표시된다
- 현재 상태가 "1개월 내역 (최신순) ∨" 형태로 표시된다
- 직접입력 선택 시 기본값은 1개월 전 ~ 오늘로 설정된다

#### 3.3 내역 표시 및 상세 이동
**사용자 스토리**: 플랫폼 사용자로서 리워드 지급 내역을 목록으로 확인하고 특정 내역을 클릭하여 상세 정보를 확인하고 싶다.

**승인 기준**:
- 모든 내역이 "USDT 리워드 지급"으로 통일 표시된다
- 선택된 기간과 정렬에 따라 실시간 필터링된다
- 조건에 맞는 내역이 없으면 "조건에 맞는 내역이 없습니다" 메시지가 표시된다
- 각 내역 항목 클릭 시 상세 페이지로 이동한다
- 마우스 포인터 스타일로 클릭 가능함을 시각적으로 표시한다

### 4. 리워드 상세 화면 (reward_history_detail.html)

#### 4.1 리워드 상세 정보 표시
**사용자 스토리**: 플랫폼 사용자로서 특정 리워드 지급 건의 상세 정보를 확인하여 거래 내역을 투명하게 파악하고 싶다.

**승인 기준**:
- URL 파라미터로 전달된 리워드 정보가 표시된다
- 상단에 녹색 원형 아이콘(달러 기호 + 숫자 6 뱃지)이 표시된다
- "입금" 제목과 "+$X" 형태의 금액이 표시된다
- "X USDT" 통화 표시와 "≈₩X" 원화 환산액이 표시된다
- 송금자(Unifi) → 수신자(나) 정보가 하나의 회색 카드에 가로 배치된다
- 거래 정보(유형, 일시, 보낸 사람)가 목록 형태로 표시된다

#### 4.2 페이지 네비게이션
**사용자 스토리**: 플랫폼 사용자로서 리워드 상세 정보 확인 후 이전 페이지로 쉽게 돌아가고 싶다.

**승인 기준**:
- 상단 좌측에 뒤로가기 버튼이 위치한다
- 하단에 "확인" 버튼이 전체 너비로 표시된다
- "확인" 버튼 클릭 시 `history.back()`으로 이전 페이지로 이동한다
- 홈 인디케이터가 하단에 표시된다

### 5. 게임 상세 화면 (game_detail_*.html)

#### 5.1 미션 진행 시스템
**사용자 스토리**: 게임 플레이어로서 각 게임의 3단계 미션을 순차적으로 완료하고 진행 상황을 시각적으로 확인하고 싶다.

**승인 기준**:
- 각 게임별 고유한 3단계 미션이 정의된다
- 순차적 미션 완료 방식 (1→2→3단계)이 적용된다
- 상태별 표시:
  - 시작 전: 회색 "시작" 버튼
  - 진행 중: 회색 "진행 중" 버튼
  - 완료: 녹색 체크 표시
- 미션 완료 시점(상태 2→3 전환)에 `Date.now()` 타임스탬프가 저장된다
- Hook & Gold는 기본 완료 상태로 고정 처리된다

#### 5.2 보상 시스템
**사용자 스토리**: 게임 플레이어로서 미션 완료에 따른 보상을 받고 하트 보상도 별도로 획득하고 싶다.

**승인 기준**:
- **메인 보상**: 3단계 모두 완료 시 2 USDT 지급
  - 보상 오버레이가 출석체크와 동일한 스타일로 표시
  - USDT 아이콘 + 그라디언트 배경으로 구성
  - LINE 알림 시뮬레이션이 함께 표시
- **하트 보상**: 게임별 독립적으로 10개 제공 (1회성)
  - 하트 받기 버튼 클릭 시 토스트 알림 표시
  - 토스트 위치: 화면 하단 (bottom: 120px), 하단 버튼 위쪽
  - 토스트 스타일: 검은색 배경(#000), 흰색 텍스트, 앱 전체 너비
  - 표시 시간: 2초간 자동 표시 후 사라짐
  - 메시지 내용: "❤️ 하트 10개 받았어요"

#### 5.3 게임 초기화 기능
**사용자 스토리**: 테스터로서 특정 게임의 진행 상태를 초기화하여 반복 테스트를 진행하고 싶다.

**승인 기준**:
- "이 게임 초기화" 버튼이 테스트 패널에 위치한다
- 클릭 시 해당 게임의 gameState와 heartState가 모두 초기화된다
- 완료 시각 정보도 함께 삭제된다
- 다른 게임의 상태에는 영향을 주지 않는다

## 기술적 요구사항

### 6.1 상태 관리
**시스템 요구사항**: 모든 게임 진행 상태와 리워드 정보가 브라우저 종료 후에도 유지되어야 한다.

**기술 명세**:
- localStorage 기반 'unifi_state' 키 사용
- JSON 형태로 상태 저장 및 로드
- 페이지 간 실시간 상태 동기화
- 데이터 무결성 보장을 위한 기본값 병합

### 6.2 애니메이션 및 UX
**시스템 요구사항**: 사용자 액션에 대한 즉각적인 피드백과 부드러운 인터랙션을 제공해야 한다.

**기술 명세**:
- 0.3초 표준 애니메이션 지속시간
- ease-out (나타남), ease-in (사라짐) 이징 적용
- 오버레이 2.5초 자동 닫기
- LINE 메시지 3.5초 표시 시간
- **토스트 알림**: 2초 표시 + 0.3초 fade 전환 효과

### 6.4 토스트 알림 시스템
**시스템 요구사항**: 비침습적이고 일관된 사용자 피드백을 제공해야 한다.

**기술 명세**:
- **CSS 클래스**: `.toast-notification` 및 `.toast-notification.show`
- **위치**: `position: fixed; bottom: 120px` (하단 버튼 위쪽)
- **크기**: `width: 100%; max-width: 480px` (앱 전체 너비)
- **스타일**: 검은색 배경(`#000000`), 흰색 텍스트(`#ffffff`)
- **애니메이션**: `opacity` 전환 (0.3초)
- **자동 닫기**: 2초 후 자동 사라짐
- **z-index**: 1000 (다른 UI 요소 위에 표시)

### 6.3 반응형 디자인
**시스템 요구사항**: 다양한 모바일 기기에서 일관된 사용자 경험을 제공해야 한다.

**기술 명세**:
- 최대 폭 480px (모바일 우선)
- CSS transition 사용 (JavaScript 애니메이션 최소화)
- CDN 이미지 사용 (obs.line-scdn.net)
- 터치 친화적 버튼 크기 및 간격

## 리스크 및 완화 전략

### 기술적 리스크
- **localStorage 용량 제한**: 정기적 데이터 정리 및 압축
