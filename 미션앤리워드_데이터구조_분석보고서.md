# 미션앤리워드 프로토타입 데이터 구조 분석 보고서

## 📋 개요
미션앤리워드는 USDT 리워드 기반 게임 플랫폼으로, 사용자가 게임 미션을 완료하여 USDT와 뽑기권을 획득할 수 있는 시스템입니다.

## 🗂️ 전체 데이터 아키텍처

### 글로벌 상태 (localStorage 'unifi_state')
```javascript
{
  usdt: number,                    // 사용자 USDT 잔고
  tickets: number,                 // USDT 뽑기권 개수
  history: [],                     // 리워드 획득 내역
  gameStates: [0,0,0,0,0,0],      // 6개 게임 미션 진행 상태 (0-4)
  heartStates: [false,false,false,false,false,false], // 하트 리워드 수령 여부
  ticketClaimed: boolean           // 게임 미션 완료 뽑기권 수령 여부
}
```

## 🎯 기능별 데이터 요구사항

### 1. 홈페이지 (index.html)

#### 필요 데이터:
- **USDT 잔고 표시**
  - `usdt`: 사용자 보유 USDT
  - `baseUsdt`: 기본 잔고 (2353.55)

- **게임 캐러셀**
  - `gameStates[]`: 6개 게임의 진행 상태
  - 게임 정보: 이름, 이미지 URL, 상세 페이지 링크

- **게임 상태 매핑**
  ```javascript
  stateConfigs = [
    { text: "0/3 미션 진행 전", color: "var(--text-sub)", bar: "0%" },
    { text: "1/3 미션 진행 중", color: "var(--text-sub)", bar: "33%" },
    { text: "2/3 미션 진행 중", color: "var(--text-sub)", bar: "66%" },
    { text: "미션 완료", color: "var(--brand-green)", bar: "100%" },
    { text: "미션 완료", color: "var(--brand-green)", bar: "100%" }
  ]
  ```

#### 데이터 플로우:
```
localStorage → getGlobalState() → renderHome() → UI 업데이트
```

### 2. 일일 미션 페이지 (daily_mission.html)

#### 필요 데이터:
- **출석 체크 시스템**
  - `currentDay`: 현재 출석일 (1-5)
  - `canAttend`: 출석 가능 여부
  - `rewards`: 일별 보상 정보

- **게임 미션 진행도**
  - `gameStates[]`: 각 게임 완료 상태
  - `ticketClaimed`: 3게임 완료 시 뽑기권 수령 여부

- **뽑기 시스템**
  - `tickets`: 보유 뽑기권 수
  - `randomReward`: 1-3 USDT 랜덤 보상

#### 출석 보상 구조:
```javascript
rewards = {
  1: { type: 'usdt', amount: 1, text: '1 USDT' },
  2: { type: 'usdt', amount: 1, text: '1 USDT' },
  3: { type: 'ticket', amount: 1, text: '뽑기권 1개' },
  4: { type: 'usdt', amount: 1, text: '1 USDT' },
  5: { type: 'ticket', amount: 3, text: '뽑기권 3개' }
}
```

### 3. 리워드 내역 페이지 (reward_history.html)

#### 필요 데이터:
- **잔고 정보**
  - `usdt`: USDT 잔고
  - `tickets`: 뽑기권 수량
  - KRW 환율: 1 USDT = 1350 원

- **거래 내역**
  ```javascript
  history = [{
    title: string,    // 리워드 타이틀
    date: string,     // 수령 일시 (YYYY-MM-DD HH:mm)
    amount: number    // USDT 금액
  }]
  ```

### 4. 게임 상세 페이지 (game_detail_*.html)

#### 필요 데이터:
- **게임 정보**
  - `GAME_INDEX`: 게임 인덱스 (0-5)
  - 게임명, 이미지, 설명

- **미션 시스템**
  - 각 게임마다 3개 미션
  - 미션 완료 시 2 USDT 보상

- **하트 리워드**
  - `heartStates[GAME_INDEX]`: 게임별 하트 수령 상태

#### 게임 상태별 UI 설정:
```javascript
uiConfig = {
  0: { // 시작 전
    m1: { txt: "0/30", cls: "ing", btnTxt: "진행 중", done: false },
    m2: { txt: "0/50", cls: "start", btnTxt: "시작", done: false },
    m3: { txt: "0/100", cls: "start", btnTxt: "시작", done: false },
    reward: { disabled: true, text: "2 USDT 받기" }
  },
  3: { // 미션 완료
    m1: { txt: "30/30", cls: "done", btnTxt: "완료", done: true },
    m2: { txt: "50/50", cls: "done", btnTxt: "완료", done: true },
    m3: { txt: "100/100", cls: "done", btnTxt: "완료", done: true },
```

## 📊 데이터 시각화

### 상태 전환도

```
게임 미션 상태 전환:
0 (시작 전) → 1 (1개 완료) → 2 (2개 완료) → 3 (모든 미션 완료) → 4 (보상 수령)

출석 체크:
Day 1-5 순환, 각 일차별 보상 지급
```

### 데이터 관계도

```
localStorage (unifi_state)
├── usdt (잔고)
├── tickets (뽑기권)
├── history[] (내역)
├── gameStates[6] (게임 진행도)
├── heartStates[6] (하트 수령 여부)
└── ticketClaimed (뽑기권 수령 여부)

주요 함수:
├── getGlobalState() - 상태 조회
├── saveGlobalState() - 상태 저장
├── addReward() - 리워드 추가
└── renderPage() - UI 렌더링
```

## 🔄 주요 비즈니스 로직

### 리워드 지급 규칙:
1. **게임 미션 완료**: 2 USDT
2. **출석 체크**: 1 USDT 또는 뽑기권
3. **뽑기**: 1-3 USDT 랜덤
4. **하트 리워드**: 게임 내 하트 10개

### 진행도 추적:
- 6개 게임 × 3개 미션 = 총 18개 미션
- 각 게임 완료 시 일일 미션 진행도에 반영
- 3개 게임 완료 시 뽑기권 3개 지급
