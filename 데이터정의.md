# 미션앤리워드 플랫폼 - 데이터 정의서

## 문서 개요

본 문서는 미션앤리워드 플랫폼의 모든 기능에서 사용되는 데이터 구조를 기획자 관점에서 정의하고, 비즈니스 로직과 연관된 데이터 요구사항을 명시합니다.

---

## 1. 전역 상태 데이터 (Global State)

### 1.1 사용자 자산 데이터

#### USDT 잔고 관리
```json
{
  "usdt": {
    "type": "Number",
    "default": 9.50,
    "validation": "≥ 0, 소수점 2자리까지",
    "display_format": "천단위 콤마, 소수점 2자리",
    "business_rule": "기본 잔고(2,353.55) + 획득 USDT로 표시"
  }
}
```

**비즈니스 요구사항**:
- 사용자가 획득한 실제 리워드 금액만 저장
- 홈 화면 표시 시에는 기본 잔고와 합산하여 보여줌
- 모든 USDT 증감은 history 배열에 기록됨
- 원화 환산 시 고정 환율 1,350원 적용

#### 럭키볼 관리
```json
{
  "tickets": {
    "type": "Integer",
    "default": 4,
    "validation": "≥ 0",
    "business_rule": "출석체크 및 게임 완성으로만 획득 가능"
  }
}
```

**비즈니스 요구사항**:
- 뽑기 실행 시 1개씩 차감
- 출석 3일차(1개), 5일차(3개), 게임 3개 완료(3개) 지급
- 0개일 때 뽑기 시도 시 에러 메시지 표시

---

### 1.2 게임 진행 상태 데이터

#### 게임 상태 배열
```json
{
  "gameStates": {
    "type": "Array[10]",
    "default": [0,1,2,3,0,0,0,0,0,0],
    "values": [0,1,2,3,4],
    "mapping": {
      "0": "미션 진행 전",
      "1": "1/3 미션 진행 중", 
      "2": "2/3 미션 진행 중",
      "3": "미션 완료 (보상 대기)",
      "4": "보상 수령 완료"
    }
  }
}
```

**비즈니스 요구사항**:
- 각 인덱스는 특정 게임에 고정 매핑
- 상태 3→4 전환 시에만 2 USDT 지급
- Hook & Gold(인덱스 3)는 기본 완료 상태 유지
- 상태 변경은 순차적으로만 가능 (되돌림 불가)

#### 게임 완료 시각 추적
```json
{
  "completionTimes": {
    "type": "Object",
    "structure": "{ 게임인덱스: timestamp }",
    "example": { "3": 1640995200000, "5": 1672531200000 },
    "business_rule": "상태 2→3 전환 시점에만 기록"
  }
}
```

**비즈니스 요구사항**:
- 프로그레스 바 표시 순서 결정에 사용
- Hook & Gold는 고정 시각으로 항상 첫 번째 유지
- 미션 완료 순서 추적을 위한 핵심 데이터
- 게임 초기화 시 해당 항목 삭제

## 3. 출석체크 시스템 데이터

### 3.1 출석 보상 정의

```json
{
  "attendance_rewards": {
    "cycle_length": 5,
    "rewards_by_day": {
      "1": {"type": "usdt", "amount": 1, "display": "1 USDT"},
      "2": {"type": "usdt", "amount": 1, "display": "1 USDT"},
      "3": {"type": "ticket", "amount": 1, "display": "USDT 럭키볼"},
      "4": {"type": "usdt", "amount": 1, "display": "1 USDT"}, 
      "5": {"type": "ticket", "amount": 3, "display": "USDT 럭키볼"}
    }
  }
}
```

**비즈니스 로직**:
- 5일 주기로 반복되는 보상 구조

- 3,5일차: 럭키볼 지급으로 간접 보상
- 사용자는 출석체크 시점에 즉시 보상 획득
